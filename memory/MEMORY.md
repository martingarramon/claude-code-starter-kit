## How to Use This File

**What to save here:**
- Current project status and next steps
- Decisions that would be expensive to re-research
- Recurring mistakes and their fixes
- Commands or configurations you look up repeatedly

**What NOT to save here:**
- Anything that belongs in code comments
- Detailed technical documentation (put that in docs/)
- Temporary debugging notes (delete after the bug is fixed)
- Information that changes every session (keep only the latest state)

**Maintenance:** Review at the start of each session. Delete anything stale.
Keep this file under 80 lines. If it grows beyond that, move detail into
separate files and link to them from here.
